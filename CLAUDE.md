# AI Triad Research ‚Äî Project Instructions

## What This Project Is

Multi-perspective research platform for AI policy & safety literature. Organizes sources through a four-POV taxonomy (accelerationist, safetyist, skeptic, cross-cutting), ingests documents, generates AI-powered summaries, and detects factual conflicts across viewpoints.

## Key Types

The `TaxonomyNode` class is defined in `AITriad.psm1` (not a separate file):
- Properties: POV, Id, Label, Description, Category, ParentId, Children, CrossCuttingRefs, Interpretations, LinkedNodes, Score

## Data Flow

1. **Ingest**: `Import-AITriadDocument` ‚Üí creates `sources/<doc-id>/` with metadata.json + snapshot.md
2. **Summarize**: `Invoke-POVSummary` ‚Üí reads snapshot, calls AI, writes `summaries/<doc-id>.json`
3. **Conflicts**: `Find-Conflict` ‚Üí reads summary's factual_claims, creates/updates `conflicts/*.json`
4. **Health**: `Get-TaxonomyHealth` ‚Üí analyzes coverage gaps across taxonomy + summaries

## Companion Modules

- `AIEnrich.psm1` ‚Äî wraps Gemini, Claude, and Groq APIs behind a unified interface
- `DocConverters.psm1` ‚Äî converts PDF, DOCX, HTML to Markdown for snapshot.md
- These are imported automatically by AITriad.psm1 at load time

## Conventions

- PowerShell 7 required. Follow PS7 rules from global CLAUDE.md (Set-StrictMode, approved verbs, Try-Catch, pipeline pollution suppression).
- Module uses Public/Private split ‚Äî new exported functions go in `Public/`, helpers in `Private/`.
- New public functions must be added to BOTH `AITriad.psm1` Export-ModuleMember AND `AITriad.psd1` FunctionsToExport.
- Electron apps all use the same stack: Electron + React + Vite. They share similar patterns for theming (Light/Dark/BKC/Auto).
- **AI prompts must be separate from code.** Never embed prompt template strings inline in hooks, stores, utilities, or components. In TypeScript apps, prompts go in `src/renderer/prompts/*.ts` as named export functions. In PowerShell, prompts go in `Prompts/*.prompt` and are loaded via `Get-Prompt`.
- Default AI model: `gemini-3.1-flash-lite-preview`. Configurable via `-Model` parameter or `$env:AI_MODEL`.
- Source doc IDs are slugs like `ai-as-normal-technology-2026`. Generated by `New-Slug` in Private/.
- `TAXONOMY_VERSION` file bump triggers CI batch re-summarization.
- **Node descriptions use genus-differentia format.** POV nodes: `"A [Category] within [POV] discourse that [differentia]. Encompasses: ... Excludes: ..."` CC nodes: `"A cross-cutting concept that [differentia]. Encompasses: ... Excludes: ..."` All prompts that create or modify descriptions enforce this pattern.
- **`node_scope`** (optional on `graph_attributes`): `"claim"` | `"scheme"` | `"bridging"` ‚Äî AIF-aligned classification of a node's role. Populated organically by `Invoke-AttributeExtraction`. Displayed as badge in `GraphAttributesPanel`.
- **`disagreement_type`** (optional on `CrossCuttingNode`): `"definitional"` | `"interpretive"` | `"structural"` ‚Äî classifies cross-POV disagreement nature. Populated by `Invoke-CrossCuttingCandidates`. Displayed as badge in `CrossCuttingDetail`.
- **`temporal_scope`** and **`temporal_bound`** on `factual_claims` in summaries ‚Äî classify when claims apply (current_state/predictive/historical/timeless). Populated organically on new summaries. `Find-Conflict.ps1` passes through when present.
- **Fallacy tiers:** All `possible_fallacies` entries have a `type` field (formal/informal_structural/informal_contextual/cognitive_bias). Deterministic lookup, 100% coverage.
- **`steelman_vulnerability`** is now a union type: `string | { from_accelerationist?, from_safetyist?, from_skeptic? }`. All consumers use `typeof` check for dual-format handling. 100% migrated to per-POV object.
- **Edge types are 7 canonical AIF-aligned types:** SUPPORTS, CONTRADICTS, ASSUMES, WEAKENS, RESPONDS_TO, TENSION_WITH, INTERPRETS. Consumer code accepts any string for backward compat. Legacy types were mapped in Phase 5; archived edges in `_archived_edges.json`.
- **Debate synthesis includes AIF argument_map.** The `argument_map` field (optional, absent in older debates) contains claims with `attack_type` (rebut/undercut/undermine) and `scheme` (COUNTEREXAMPLE/DISTINGUISH/etc.). Consumers must handle absence via optional chaining. `Find-Conflict.ps1` also accepts optional `attack_type`, `target_claim`, `counter_evidence` on conflict instances.
- **Debate agents use BDI-structured taxonomy context.** `formatTaxonomyContext` (in `taxonomy-editor/src/renderer/utils/taxonomyContext.ts`) groups POV nodes into Beliefs (Data/Facts), Values (Goals/Values), and Reasoning (Methods/Arguments) sections. Vulnerabilities and cross-cutting concerns are separate sections. Debate synthesis includes `bdi_layer` and `resolvability` fields (optional ‚Äî absent in pre-migration debates).
- **Incremental argument network.** After each debater's turn, a background AI call extracts claims and maps relationships to prior claims (`debate.argument_network`). Validated by word overlap and reference checks. Fed to the moderator for cross-respond selection. Prompt in `prompts/argumentNetwork.ts`.
- **Per-debater commitment tracking.** Derived from the argument network: asserted, conceded, challenged claims tracked in `debate.commitments`. Injected into debater prompts via `formatCommitments()` with a consistency rule ("do not silently contradict prior assertions").
- **Preference resolution in synthesis.** Each disagreement is evaluated for which argument prevails, by what criterion (`empirical_evidence`, `logical_validity`, `source_authority`, `specificity`, `scope`), with rationale. Rendered as "Resolution Analysis" in synthesis cards. `SynthesisResult.preferences` field.
- **Debate Harvest.** "Harvest" button (after synthesis) opens a dialog to promote debate findings into the taxonomy. Five item types: conflicts ‚Üí `conflicts/*.json`, steelman refinements ‚Üí node `steelman_vulnerability`, debate refs ‚Üí node `debate_refs`, preference verdicts ‚Üí conflict `verdict` field. All items user-reviewed with AI-generated descriptions validated before display. Harvest manifests in `ai-triad-data/harvests/`.
- **Debate-specific AI model.** Each debate can override the global model via `debate_model` on `DebateSession`. Set in New Debate dialog, persisted and restored on load. `getConfiguredModel()` checks debate model first.
- **Fact-check with web search.** `generateTextWithSearch` uses Gemini's `google_search` tool for grounded fact-checking. Web evidence stored in metadata and viewable via "Show Web Evidence" button on fact-check cards.
- **`debate_refs`** (optional on `PovNode` and `CrossCuttingNode`): array of debate session IDs that referenced this node. Added by harvest.
- **POVer Chat** is a 1-on-1 conversation with a single POVer (distinct from multi-agent Debate). Three modes: `brainstorm` (exploratory, high-creativity), `inform` (pedagogical, grounded), `decide` (analytical, Socratic). Mode is switchable mid-chat via header pills. Chat sessions persist in `chats/chat-<id>.json`. Types in `types/chat.ts`, store in `hooks/useChatStore.ts`, prompts in `prompts/chat.ts`. Same taxonomy grounding and BDI context as debates, but lighter ‚Äî no clarification phase, no multi-agent coordination.
